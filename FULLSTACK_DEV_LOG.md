- 属于类型检查器的正确性修复，提升错误诊断质量

  - 索引有效 → 返回对应元素类型
- **ADT 类型**：明确提示 ADT 类型需通过模式匹配访问字段，而非直接 `.field`
- **其他类型**：给出精确的"不支持字段访问"错误和提示

### ⚙️ 后端：实现原生后端浮点/字符串常量加载

**为什么选这个？**
- 依赖 `backend_native_label_patch` 已在上轮完成，依赖链畅通
- 原生后端当前只能处理整数，浮点和字符串常量都是 `pass` 占位
- x86_64 指令编码器（`x86_64.py`）已有 `movsd_reg_imm` 和 `lea_reg_rip`，基础设施就绪
- 中等难度，是原生后端从"整数原型"迈向"可用"的重要一步
- 上轮下一步计划中明确提到优先考虑此任务

**实现详情**：
- **常量收集优化**（`_collect_constants`）：增加去重逻辑，相同浮点值/字符串只存一份到数据段，构建 `_float_const_map` 和 `_string_const_map` 快速查找表
- **浮点常量加载**（`LIRLoadConst` float 分支）：
  - 调用 `e.movsd_reg_imm(reg, 0)` 发射 `movsd xmm, [rip + offset]` 指令
  - 记录 `data_fixups` 项：`(func_name, fixup_offset, data_offset, "float")`
- **字符串常量加载**（`LIRLoadConst` string 分支）：
  - 调用 `e.lea_reg_rip(reg, 0)` 发射 `lea reg, [rip + offset]` 指令
  - 记录 `data_fixups` 项：`(func_name, fixup_offset, data_offset, "string")`
- **ELF 阶段回填**（`_generate_elf`）：
  - 计算数据段虚拟地址（按页对齐）
  - 遍历 `data_fixups`，计算 RIP-relative 偏移 = 目标地址 - (rel32位置 + 4)
  - 使用 `struct.pack_into` 回填到代码段

**结果**: ✅ 成功，测试全部通过

### ⚙️ 后端：实现原生后端两阶段汇编与标签回填

**为什么选这个？**
- 优先级 92（后端最高），是原生后端所有其他功能的基石
- 没有标签回填，控制流（jump/branch）全部跳到错误地址
- emitter 层（`x86_64.py`）已有 `patch_rel32` 等基础设施，缺的是调度逻辑
- 是后续寄存器分配、栈帧管理、ABI 等任务的前置依赖

**实现详情**：
- 在 `_compile_body` 中引入两阶段汇编架构
- **第一阶段**：发射指令，同时维护 `label_offsets`（标签名→偏移）和 `jump_fixups`（待回填跳转列表）
  - `LIRLabel`：记录 `e.current_offset()` 到 `label_offsets`
  - `LIRJump`：发射 `jmp_rel32` 占位，记录 `(offset, target, "jmp")`
  - `LIRBranch`：发射 `test + jne + jmp`，分别记录两个回填项
- **第二阶段**：遍历 `jump_fixups`，调用 `e.patch_rel32(offset, target_offset)` 回填

**结果**: ✅ 成功，测试全部通过
