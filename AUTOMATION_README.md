# Nova 自动化改进系统

## 架构

```
┌─────────────────────────────────────────┐
│         每小时定时任务（调度器）            │
│  AUTO_IMPROVE_LOG.md ← 状态中心           │
│         ↓                                 │
│  读取日志 → 分析当前状态 → 分配任务         │
│         ↓                                 │
│  ┌─────┼─────┼─────┬─────┬─────┐         │
│  │  任务1  任务2  任务3  ...  任务N  │      │
│  │ (并行 ≤3 个 agent)                  │      │
│  └─────┼─────┼─────┴─────┴─────┘         │
│         ↓                                 │
│  验证测试 → 更新日志 → 提交推送             │
└─────────────────────────────────────────┘
```

## 日志文件（状态中心）
- `/workspace/nova/AUTO_IMPROVE_LOG.md` — 唯一的"记忆"
- 记录：已完成项、待改进项、发现的问题、改进历史
- 每次任务读取它来决定下一步做什么
- 每次任务完成后更新它

## 任务队列（在日志中维护）
日志的"待改进项"就是任务队列，按 P0→P3 排序。
每次任务取队列顶部 2-3 项执行，完成后标记为已完成。
发现新问题时追加到队列中。

## 协作机制
- 所有任务共享同一个日志文件（唯一状态源）
- 任务完成后立即更新日志，下一个任务能看到最新状态
- git 提交作为自然锁：如果文件被修改，先 pull 再 push
